P3.  Anagrams [20 points]

Two strings, a and b, are called anagrams if they contain all the same characters in the same frequencies. For example, the anagrams of LAB are LAB, ALB, ABL, LBA, BAL, and BLA.

Now you need to write a program named  Anagrams.java that takes two strings s1 and s2 as input

from user (input from console and not as arguments) and prints "Anagrams" if s1 and s2 are case-insensitive anagrams  and prints "Not Anagrams" if they are not.

Sample test case:

$ javac Anagrams.java

$ java Anagrams

AnagraM

margaNa

Anagrams

P5.  Get Synchronized  [30 points]

In this task we will try to simulate a Producer Consumer Market. For our purposes Market is a single member variable int totalStock of class ProducerConsumer initialized to 0. The same class has the main method, in which two threads called as producer and consumer are declared.

A producer increments totalStock by one (totalStock++) in a for loop for 100000 iterations. A consumer does the same except that it decrements totalStock by one (totalStock--).

After declaring the threads, they should be started using start() method of Thread class and the main program should wait for both of them to finish execution (use join() method of a Thread class to achieve this). After both of the threads have completed execution, print the value of totalStock to stdout.

Can you guess this output?

Our intuition suggests that output should be 0, because 100000 units will be produced and 100000 units will be consumed. Does the output match your expectation? Try running it for at least 5 times and observe the results.

Submit ProducerConsumer.java in the submission.

If the results do not match your expectations, what do you think might be the cause for it?

Copy paste the ProducerConsumer.java and change class and file name to ProducerConsumerFix and ProducerConsumerFix.java and make changes such that finally output (final value of totalStock) is 0 always.

Submit ProducerConsumerFix.java in the submission.
